# Day 12 – Opportunity Identification (Category Overspend)

## What was built
- Opportunity identification service layer
- Category overspend detection logic
- Conservative savings estimation
- Human-readable coaching suggestions
- Isolated testing of opportunity logic

## Key decisions
- Relative (percentage-based) overspend detection
- Gentle savings recommendations (10–20%)
- No judgmental language
- Opportunities treated as suggestions, not rules

## What I learned
- How to translate patterns into actions
- Why explainability matters for trust
- How coaching logic differs from analytics
